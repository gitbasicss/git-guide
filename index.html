Команда успішно вирішила їх, об'єднавши код усіх учасників без втрати інформації. --- ## 📚 Короткий довідник, який ми розробили | Команда | Опис | | :--- | :--- | | `git init` | Ініціалізація локального репозиторію | | `git clone ` | Клонування віддаленого репозиторію на ПК | | `git checkout -b ` | Створення нової гілки та перехід на неї | | `git status` | Перевірка стану робочого дерева | | `git add .` | Додавання всіх змін до індексу (staging) | | `git commit -m "msg"` | Фіксація змін із коментарем | | `git pull origin main` | Отримання актуальних змін із сервера | | `git push origin `| Відправка локальних коммітів на GitHub | --- ## 🚀 Деплой (GitHub Pages) Результат нашої командної роботи — працюючий веб-сайт, який автоматично розгорнуто на безкоштовному хостингу. 🔗 **Посилання на живий проєкт:** [Переглянути Git Guide на GitHub Pages](https://<назва_організації>.github.io/git-guide/) --- *Виконано в рамках 1-годинного командного спринту з оцінюванням кожного етапу.*
